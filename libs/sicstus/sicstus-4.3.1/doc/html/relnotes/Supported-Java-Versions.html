8.1 Supported Java Versions

Jasper requires at least Java 1.5 to run. Except under Windows the full development kit, not just the JRE, is needed. c [PM] 4.3 irrelevant these days: Jasper does not work with Visual J++ or Visual Café. Unless indicated otherwise, you can download the JDK from http://java.sun.com.

Except where indicated, Jasper is supported for Java 1.5 or later.

For some platforms, Jasper is only supported under the following conditions:

Mac OS X
Using Jasper from Java may require that DYLD_LIBRARY_PATH be set up so that Java can find the SICStus runtime library. That is, you may need to set DYLD_LIBRARY_PATH to the location of the SICStus runtime libsprt4-3-1.dylib.

On OS X you can either use the legacy Java 6 from Apple, or Java 7 or later from http://java.sun.com.

On OS X, embedding Java 7 or later in SICStus, e.g. by using library(jasper), may fail, possibly with a dialog about the need for a Java SE 6 runtime. This seems to be caused by missing information in the section ‘JVMCapabilities’ of the file Info.plist in the Java installation. As of April 2014 there were no official information from Oracle about this but several suggested workaround can be found on the net.
